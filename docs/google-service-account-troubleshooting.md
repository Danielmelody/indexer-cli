# Google Service Account Troubleshooting

## "Not enough private keys in PEM" Error

### Root Cause

This error occurs when the service account JSON file contains a private key in **PKCS#1 format** (`BEGIN RSA PRIVATE KEY`) instead of the required **PKCS#8 format** (`BEGIN PRIVATE KEY`).

### Expected Format

**Correct (PKCS#8)**:
```
-----BEGIN PRIVATE KEY-----
MIIEvQIBADANBgkqhkiG9w0BAQEF...
-----END PRIVATE KEY-----
```

**Incorrect (PKCS#1)**:
```
-----BEGIN RSA PRIVATE KEY-----
MIIEpAIBAAKCAQEA2Z1JqSy9f...
-----END RSA PRIVATE KEY-----
```

### Why This Happens

1. **Old Creation Method**: Key was created using an older API or tool
2. **Manual Editing**: Key was manually converted or edited
3. **Third-Party Tools**: Key was generated by non-Google tools
4. **Test/Mock Data**: Using test data instead of real Google-generated keys

### Solution: Create a New Key

**Google Cloud Console Method** (Recommended):

1. **Visit Service Accounts**:
   ```
   https://console.cloud.google.com/iam-admin/serviceaccounts
   ```

2. **Select Your Project**:
   - Make sure you're in the correct project
   - Look for your service account in the list

3. **Create New Key**:
   - Click on the service account name
   - Go to the "KEYS" tab
   - Click "ADD KEY" → "Create new key"
   - **IMPORTANT**: Select "JSON" format (NOT P12)
   - Click "CREATE"

4. **Verify the Downloaded File**:
   ```bash
   cat ~/Downloads/your-key.json | jq -r '.private_key' | head -1
   ```

   Should show: `-----BEGIN PRIVATE KEY-----` ✓

   NOT: `-----BEGIN RSA PRIVATE KEY-----` ✗

### What Google Actually Generates

When you create a JSON service account key through the official Google Cloud Console:

- ✅ **Always** uses PKCS#8 format (`BEGIN PRIVATE KEY`)
- ✅ **Never** uses PKCS#1 format (`BEGIN RSA PRIVATE KEY`)
- ✅ Compatible with all Google APIs and client libraries

If you see `BEGIN RSA PRIVATE KEY`, the key was **NOT** created through the standard Google Cloud Console JSON download process.

### Manual Conversion (Last Resort)

If you absolutely cannot get a new key from Google, you can convert PKCS#1 to PKCS#8:

```bash
# Extract the private key from JSON
cat your-file.json | jq -r '.private_key' > old_key.pem

# Convert PKCS#1 to PKCS#8
openssl pkcs8 -topk8 -inform PEM -outform PEM \
  -in old_key.pem -out new_key.pem -nocrypt

# Update the JSON file
# Manually replace the "private_key" field value with the content of new_key.pem
# Make sure to escape newlines as \n in the JSON
```

**Warning**: Manual conversion is error-prone. **Always prefer creating a new key** from Google Cloud Console.

### Add to Google Search Console

After getting a valid key, remember to add the service account to Google Search Console:

1. Visit: https://search.google.com/search-console
2. Select your website property
3. Settings → Users and permissions → Add user
4. Email: `your-sa@project-id.iam.gserviceaccount.com` (from JSON)
5. Permission: **Owner** (required for Indexing API)

## Common Questions

**Q: Why does Google use PKCS#8?**

A: PKCS#8 is the modern standard that includes algorithm information within the key, making it more versatile and compatible with various APIs.

**Q: Can I use P12 format instead?**

A: P12 is deprecated by Google. Always use JSON format for new keys.

**Q: How do I know if my key is from Google?**

A: Real Google-generated JSON keys always have:
- `BEGIN PRIVATE KEY` (PKCS#8 format)
- Valid `client_email` ending in `.iam.gserviceaccount.com`
- `auth_uri` pointing to `accounts.google.com`
- `token_uri` pointing to `oauth2.googleapis.com`

**Q: My key still doesn't work after creating a new one**

A: Check these:
1. Wait 3-5 minutes after enabling the Indexing API
2. Ensure service account is added to Search Console as **Owner**
3. Verify you're using the latest JSON file you just downloaded
4. Make sure the file hasn't been edited or corrupted
